# ProvenQuant Trader

A self-hosted, FinML-based algorithmic trading system. Run trading strategies powered by machine learning models on your own machine — with a single command.

> **Not ready to self-host?** Use the cloud version at [provenquant.com](https://www.provenquant.com)

---

## What Is This?

ProvenQuant Trader is an open-source, Docker-based trading bot platform designed for users who are comfortable working with code and want full control over their trading system. It is built around Financial Machine Learning (FinML) techniques and can run standalone or integrate with the [ProvenQuant](https://www.provenquant.com) platform to pull live ML signals.

**Key characteristics:**

- **FinML-based** — strategies powered by triple barrier labeling, regime detection, and meta-labeling models
- **Self-hosted** — runs entirely on your own machine with Docker Compose
- **Scalable** — designed to scale up easily when you're ready
- **Extensible** — write custom strategies by subclassing a single base class
- **Simulation mode** — paper trade safely with Redis-backed live positions and Postgres trade history before going live

## Quick Start

The fastest way to get started is a single command:

```bash
./setup.sh
```

This will:
1. Check that Docker is installed and running
2. Generate a `.env` file with secure random secrets
3. Build and start all services via Docker Compose
4. Run database migrations automatically

That's it — the system is running.

## Prerequisites

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) (includes Docker Compose v2)
- A machine with at least 2 GB of RAM available for Docker

## Services

After setup, the following services are running:

| Service | Description | Default Port |
|---|---|---|
| `backend` | FastAPI REST API | 8001 |
| `backend-worker` | Celery workers (strategy execution) | — |
| `backend-beat` | Celery beat (scheduled data collection) | — |
| `trader` | WebSocket data fetcher + strategy dispatcher | — |
| `frontend` | Web UI dashboard | 3002 |
| `postgres` | Database (positions, ticks, strategies) | — |
| `redis` | Live position state + message broker | — |
| `rabbitmq` | Task queue for Celery | — |

## Architecture

```
BinanceDataFetcher  (WebSocket)
    │  on closed bar
    ▼
StrategyExecuterManager
    │  dispatches Celery task per strategy (Redis lock: 1 concurrent execution per ID)
    ▼
run_strategy task  (Celery worker)
    │
    ├─ StrategyExecuter.execute(tick) → TradeSignal | None
    │
    └─ TradeExecuter → PaperTradeAdapter → Redis (live positions) + Postgres (closed positions)

DataCollector (Celery beat, every 1 min)
    └─ Redis tick buffers → Postgres ticks table
       └─ ProvenQuantDataCollector also POSTs heartbeat to main API
```

## Writing a Custom Strategy

Create a new file in `backend/strategies/` and subclass `StrategyExecuter`:

```python
from app.services.data_fetcher import Subscription, TickData
from app.services.strategy_executer import StrategyExecuter, TradeSignal, SignalSide

class MyStrategy(StrategyExecuter):
    @property
    def id(self) -> str:
        return "my_unique_strategy_id"

    @property
    def subscriptions(self) -> list[Subscription]:
        return [Subscription(asset_slug="btcusdt", timeframe="30m")]

    async def execute(self, tick: TickData) -> TradeSignal | None:
        # your logic here
        ...
```

Then register it:

```bash
sudo docker compose exec backend python tasks.py start-trader \
  --strategy strategies.my_strategy.MyStrategy
```

See `backend/strategies/` for working examples:
- `example_rsi.py` — simple RSI strategy
- `example_pair_trade.py` — pair trading
- `multi_blade_meta_labeling_model_strategy.py` — full FinML pipeline

## Integrating ProvenQuant ML Signals

To pull live predictions from the ProvenQuant platform, add your API key to `.env`:

```env
PROVENQUANT_API_URL=https://www.provenquant.com/api
PROVENQUANT_API_KEY=your_key_here
```

Then call `DatasetFetcher.fetch_predictions()` inside your strategy's `execute()` method.

## Key Commands

```bash
# Start all services
./setup.sh

# Stop all services
./stop.sh

# Check service status
./status.sh

# Run database migrations manually
sudo docker compose exec backend alembic upgrade head

# Check paper trade balance
sudo docker compose exec backend python tasks.py paper-balance

# Reload strategies at runtime (no restart needed)
sudo docker compose exec redis redis-cli -a $REDIS_PASSWORD PUBLISH strategy:config_changed ""

# Tail worker logs
sudo docker compose logs -f backend-worker
```

## Configuration

All configuration is done through the `.env` file (auto-generated by `setup.sh`). Key variables:

| Variable | Description |
|---|---|
| `API_KEY` | Bearer token for protected API endpoints |
| `SERVER_SECRET` | Encryption key for stored exchange credentials |
| `POSTGRES_PASSWORD` | Database password |
| `REDIS_PASSWORD` | Redis password |
| `PROVENQUANT_API_KEY` | ProvenQuant API key (optional) |
